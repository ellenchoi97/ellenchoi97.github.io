Air Race Over UCSD

Unity C# Oculus Rift Leap Motion

This project was the fourth assignment for UCSD's 3D User Interation class and it was a program where the user is inside an "aircraft" and they must control the aircraft to reach all the checkpoints of the given race track. The purpose of the project was to become familiar with coding with the Leap Motion, implementing efficient controls using the Leap Motion, and implementing wayfinding methods.

Project Rubric and Directions

Goals for this Project:

  1. Write a parser to get the coordinates for the checkpoints for the race track and create a visual representation of the track by drawing the checkpoints.
  2. A checkpoint is reached when the aircraft collides with the checkpoint. Implement a visual indication that a checkpoint has been reached (different colored checkpoints).
  3. Start at the first checkpoint. Display a countdown timer and do not allow the aircraft to move until the countdown ends.
  4. Display a stopwatch once the countdown ends. Stop the stopwatch when all checkpoints have been reached in order.
  5. Implement controls for the aircraft using only data from the Leap Motion.
    • The aircraft cannot teleport anywhere but it may stop and start instantaneously.
    • Auto pilot is not allowed.
  6. Implement two different visual methods for wayfinding (arrow indicator and text showing distance remaining).
  7. Display the wayfinding visuals in different coordinate spaces (aircraft coordinates and head/Oculus Rift's coordinates).
  8. If the aircraft collides with the floor or a building, move back to the last successfully cleared checkpoint. Implement a countdown before the aircraft can move again.

Aircraft Controls:

  • Thumb Up (Left): Turn to the left
  • Thumb Up (Right): Turn to the right
  • Palm Up (Left or Right, not Both): Move up
  • Palm Down (Left or Right, not Both): Move down
  • Fist (Left, Right, or Both): Move forward
  • Index Finger Extended (Left, Right, or Both): Move backward

* Movements can be combined (ex. left thumb up and right palm up lets the user move up and turn left)
* If the user has a fist in one hand and an extended index finger in the other, no movement will occur (Cannot move forward and backward).
